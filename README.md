## üéØ Overview

**CELRIX** is a next-generation in-memory cache database built from the ground up for maximum throughput and minimal latency. Designed as a drop-in Redis replacement, CELRIX achieves breakthrough performance through innovative lock-free architecture and a custom binary protocol.

### Key Metrics

| Metric | Target |
|--------|--------|
| **Throughput** | 1M+ ops/sec |
| **P99 Latency** | <5ms |
| **Protocol Efficiency** | 22-byte frames (vs Redis RESP 35+ bytes) |

## ‚ú® Features

### Core Cache (Phase 1)
- ‚ö° **VCP Protocol** - Custom binary protocol with 22-byte frames
- üîë **Basic Commands** - PING, GET, SET, DEL, EXISTS
- ‚è∞ **TTL Support** - Automatic key expiration with background cleaner
- üìä **Metrics** - Operations counters and latency measurement
- üñ•Ô∏è **CLI Client** - Built-in test client

### High Concurrency (Phase 2)
- üßµ **Multi-core Workers** - One acceptor + N worker executors pinned to cores
- üîì **Lock-Free Data Structures** - RCU HashMap for zero-contention reads
- üì¨ **MPMC Queues** - Bounded command routing between tasks
- üö´ **Zero-Allocation Hot Path** - Pooled buffers and pre-allocated frames

### Advanced Performance (Phase 3)
- üîå **Connection Multiplexing** - Handle thousands of concurrent connections
- üì¶ **Request Pipelining** - Batch multiple commands per round-trip
- üî¢ **Extended Commands** - SCAN, MGET/MSET, INCR/DECR
- üßπ **Smart Eviction** - LRU/LFU policies with configurable memory limits

### AI & Vector Add-ons (Phase 4)
- üß† **Embedding Store** - Store and retrieve vector embeddings
- ‚ö° **SIMD Acceleration** - AVX2-optimized similarity search
- üîç **Semantic Caching** - Query by similarity with configurable thresholds
- üîó **External Integration** - HTTP/gRPC hooks for LLM/embedding services

### Production Ready (Phase 5)
- üíæ **Persistence** - RDB-style snapshots and AOF write-ahead logging
- üìà **Observability** - Prometheus metrics, detailed tracing
- ‚öôÔ∏è **Admin API** - Hot configuration, policy changes, health monitoring
- üîí **Battle-Tested** - Fuzzing, chaos testing, load benchmarks

## üöÄ Quick Start

### Prerequisites

- Rust 1.75+ (with cargo)
- Linux/macOS (Windows support planned)

### Build from Source

```bash
# Clone the repository
git clone https://github.com/YASSERRMD/celrix.git
cd celrix

# Build release binary
cargo build --release

# Run the server
./target/release/celrix-server --port 6380

# In another terminal, use the CLI client
./target/release/celrix-cli --host 127.0.0.1 --port 6380
```

### Basic Usage

```bash
# Connect to CELRIX
celrix-cli> PING
PONG

# Set a value (with optional TTL in seconds)
celrix-cli> SET mykey "Hello, CELRIX!" 3600
OK

# Get a value
celrix-cli> GET mykey
"Hello, CELRIX!"

# Check existence
celrix-cli> EXISTS mykey
1

# Delete a key
celrix-cli> DEL mykey
1
```
